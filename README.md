## Why This Repo Exists

Pinout Plus is designed to help users work with pinouts, wiring references, diagrams, connectors, and related technical content in a cleaner and more manageable way.

This repository exists so users have a clear, consistent place to:

- Report bugs and broken site behavior
- Flag incorrect, missing, or confusing technical information
- Share reproduction steps for layout or workflow issues
- Track ongoing fixes and known problems
- Suggest improvements tied to real usage problems

## What Belongs In An Issue

Please open an issue when you run into something that is clearly wrong, broken, missing, or unexpectedly hard to use.

Examples include:

- A page fails to load
- A diagram, connector, or pinout appears incorrect
- Search, navigation, filtering, or editing behaves unexpectedly
- Layout breaks on desktop, tablet, or mobile
- Images, previews, or downloads fail to render
- A save, import, login, or sharing flow does not complete correctly
- Accessibility problems make the interface difficult to use
- Error messages appear during normal use

## What To Include In A Good Report

The more specific the report, the faster it can be reproduced and fixed.

Include these details whenever possible:

| Detail | Why It Helps |
| --- | --- |
| Short issue title | Makes the problem easy to scan and track |
| Page URL | Shows exactly where the issue occurred |
| Expected result | Clarifies intended behavior |
| Actual result | Explains the failure clearly |
| Reproduction steps | Helps confirm and isolate the bug |
| Browser and version | Useful for browser-specific issues |
| Operating system | Helps spot platform differences |
| Device type | Important for responsive or touch issues |
| Screenshots or recordings | Speeds up diagnosis |
| Error text | Preserves exact failure details |
| Date and time | Helps match logs or recent changes |

## Before You Open A Ticket

Please take a quick pass through this checklist first:

- Search existing issues to avoid duplicates
- Confirm the problem can be reproduced
- Double-check the page URL
- Include enough detail for someone else to repeat the issue
- Add screenshots when the problem is visual

## Best Reports For Pinout Plus

The most helpful tickets usually describe:

- What part of the site was being used
- What action triggered the issue
- Whether it happens every time or only sometimes
- Whether the problem affects diagrams, connectors, accounts, permissions, or saved content
- Whether the issue was found on the live site or a local/dev environment

## Scope Of This Repository

This repository is intended to support issue tracking and user feedback for the main Pinout Plus platform. It is the documentation and reporting layer for the project, not the primary application codebase.

If you are reporting a problem, clarity matters more than technical depth. A simple, accurate bug report is far more useful than a vague one.
